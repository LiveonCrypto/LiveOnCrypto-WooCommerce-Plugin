Accept cryptocurrency payments in WooCommerce through LiveOnCrypto with server-side payment intents, signed webhooks, Blocks checkout support, and HPOS-compatible order metadata.

== Description ==

LiveOnCrypto for WooCommerce adds a cryptocurrency payment method to WooCommerce. Merchants enter their own LiveOnCrypto public key, secret key, and webhook signing secret in WooCommerce settings; no merchant credentials are committed in the plugin. The plugin uses the hard-coded production API endpoint and widget script URL.

Orders remain pending or on-hold while the shopper completes payment. During checkout the plugin creates a server-side LiveOnCrypto payment intent when credentials are available, stores the returned payment identifier on the WooCommerce order, and renders the LiveOnCrypto widget on the order-pay page. The plugin marks an order paid only after receiving a LiveOnCrypto webhook with a valid signature, fresh timestamp, matching fiat amount, and matching fiat currency. Payment metadata is stored on the WooCommerce order for support and reconciliation.

= Minimum versions =

* PHP: 8.0 or newer.
* WordPress: 6.0 or newer; tested up to 6.8.
* WooCommerce: 7.1 or newer; tested up to 9.0.

= Compatibility =

* High-Performance Order Storage (HPOS): supported and declared compatible.
* Classic checkout: supported.
* WooCommerce Blocks checkout: supported through the WooCommerce Blocks payment method integration. The classic gateway remains the source of truth for order creation, payment intent creation, and order-pay redirects.

== Installation ==

1. Download the release ZIP containing the `liveoncrypto-woocommerce` plugin directory.
2. In WordPress admin, go to Plugins > Add New > Upload Plugin.
3. Upload the ZIP, click Install Now, and activate the plugin.
4. Confirm WooCommerce is installed and active.
5. Go to WooCommerce > Settings > Payments.
6. Select LiveOnCrypto and complete the merchant setup steps below.

You can also install manually by uploading the `liveoncrypto-woocommerce` directory to `/wp-content/plugins/` and activating it from Plugins.

== Merchant setup ==

1. Install and activate LiveOnCrypto for WooCommerce.
2. Go to WooCommerce > Settings > Payments and enable the LiveOnCrypto payment method.
3. Enter your LiveOnCrypto public client key. It must begin with `pk_live_` or `pk_test_`.
4. Enter your LiveOnCrypto secret client key. It must begin with `sk_live_` or `sk_test_`.
5. Choose the pending order status behavior (`pending` or `on-hold`) and optionally set payment expiration behavior.
6. Copy the webhook endpoint URL shown in the LiveOnCrypto gateway settings.
7. Paste that webhook URL into your LiveOnCrypto dashboard webhook settings.
8. Copy the webhook signing secret generated by LiveOnCrypto. It must begin with `whsec_`.
9. Paste the webhook signing secret into the LiveOnCrypto gateway settings.
10. Save changes.
11. Use the connection test and then place a test checkout order end-to-end.

== Frequently Asked Questions ==

= Does the plugin custody funds? =

No. The plugin connects WooCommerce to LiveOnCrypto payment flows and webhook notifications. It does not custody merchant or customer funds.

= Are private wallet keys stored? =

No. The plugin does not ask for, store, or use private wallet keys. It stores only the merchant-configured LiveOnCrypto API keys, webhook signing secret, gateway settings, and order payment metadata needed for reconciliation.

= When is an order marked paid? =

An order is marked paid only after a signed LiveOnCrypto webhook is verified within the replay window and the webhook reports a `payment.paid` event with `paid` status plus the expected fiat amount and currency for the WooCommerce order. Overpayment events can also complete the order when the fiat amount due is satisfied and the currency matches.

= What happens if a webhook is delayed? =

The order remains pending or on-hold until the verified webhook arrives. Merchants can review LiveOnCrypto diagnostics and order notes while waiting.

= Does this support HPOS? =

Yes. The plugin declares compatibility with WooCommerce High-Performance Order Storage and uses WooCommerce order APIs for payment metadata.

= Does this support Blocks checkout? =

Yes. The plugin declares WooCommerce cart and checkout Blocks compatibility and registers a Blocks payment method for LiveOnCrypto. The Blocks integration displays the configured title and description, then lets WooCommerce use the classic gateway for payment processing and the order-pay widget flow.

== Uninstall ==

By default, uninstalling the plugin preserves WooCommerce order payment records and LiveOnCrypto order metadata for accounting, refunds, chargeback/dispute research, and support.

Plugin settings can be deleted on uninstall only when the merchant explicitly enables the cleanup setting in WooCommerce > Settings > Payments > LiveOnCrypto before uninstalling. Order payment records are not deleted by the plugin uninstall routine.

Before tagging a production release, verify:

* Test PHP 8.0, 8.1, 8.2, and 8.3.
* Test current WordPress.
* Test current WooCommerce.
* Test HPOS on.
* Test HPOS off.
* Test classic checkout.
* Test Blocks checkout.
* Test key rotation for public key, secret key, and webhook signing secret.
* Test webhook delivery with pretty permalinks.
* Test webhook delivery with plain permalinks.
* Test duplicate webhook events.
* Test invalid webhook signatures.
* Test stale webhook timestamps.
* Test amount mismatch webhooks.
* Test currency mismatch webhooks.
* Test expired payment behavior for on-hold, failed, and cancelled orders.
